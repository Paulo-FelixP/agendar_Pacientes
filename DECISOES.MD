# Decisões

## Principais decisões do Projeto:

- Utilização do Spring Boot para construção da API REST.
- Separação da aplicação em camadas (Controller, Service, Repository e DTO).
- Utilização de DTOs para entrada de dados, evitando exposição direta das entidades.
- Utilização de Bean Validation para validação dos dados recebidos.
- Front-end desenvolvido com HTML, CSS e JavaScript puro, consumindo a API via requisições HTTP.
- Médicos definidos previamente na aplicação, permitindo foco na gestão de pacientes e agendamentos.

---

## Prioridade:

- Organização do código.
- Separação de responsabilidades.
- Simplicidade na utilização.
- Validação dos dados enviados pelo usuário.
- Integração entre front-end e back-end.

---

## Ficou de fora:

- Sistema de autenticação.
- Controle de permissões.
- Persistência em banco de dados relacional externo.
- Testes automatizados.
- Paginação das consultas.
- Documentação automática da API (Swagger/OpenAPI).

---

## Utilização de IA

Foi utilizada IA como apoio para:

- Consulta sobre anotações e recursos do Spring Boot.
- Esclarecimento de dúvidas relacionadas ao Bean Validation.
- Auxílio na integração entre o front-end e o back-end.
- Sugestões para implementação de funcionalidades em JavaScript.
